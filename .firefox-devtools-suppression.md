# Firefox installHook.js.map Fehler beheben

## Problem
Firefox zeigt folgende Fehler in der Konsole:
```
[404] /installHook.js.map
Error: Promised response from onMessage listener went out of scope chrome.js:129:30
Source map error: request failed with status 404
```

## Ursache
Eine Browser-Extension (wahrscheinlich React/Vue DevTools oder ähnlich) injiziert `installHook.js` in die Seite, aber die Source Map existiert nicht.

## Lösungen

### Lösung 1: Extensions temporär deaktivieren (empfohlen für Development)

1. Firefox öffnen
2. `about:debugging#/runtime/this-firefox` in die Adressleiste eingeben
3. Unter "Erweiterungen" jede Extension einzeln deaktivieren:
   - React Developer Tools
   - Vue.js devtools
   - Redux DevTools
   - Andere Developer-Extensions
4. Seite neu laden und prüfen, ob Fehler verschwunden ist

### Lösung 2: Source Map Fehler in Firefox unterdrücken

1. Firefox DevTools öffnen (F12)
2. Einstellungen öffnen (Zahnrad-Icon oben rechts)
3. Unter "Erweiterte Einstellungen":
   - ☑ "Source-Map-Fehler anzeigen" **deaktivieren**

### Lösung 3: Browser-Kontext für Development verwenden

Erstelle ein separates Firefox-Profil nur für Development ohne Extensions:

```bash
firefox -P -no-remote
```

Dann ein neues Profil "Development" erstellen ohne Extensions.

### Lösung 4: Source Map Mock erstellen (nicht empfohlen)

Falls du die Extension unbedingt brauchst, kannst du eine leere Source Map erstellen:

```bash
echo '{"version":3,"sources":[],"names":[],"mappings":""}' > public/installHook.js.map
```

## Verifizierung

Nach Anwendung einer Lösung:
1. Browser komplett schließen
2. Firefox neu starten
3. `npm run dev` ausführen
4. http://localhost:4321/ öffnen
5. DevTools öffnen (F12) und Konsole prüfen

Die `[404] /installHook.js.map` Meldungen sollten verschwunden sein.

## Status

Dieser Fehler beeinträchtigt **nicht** die Funktionalität der Website. Er ist nur störend in den DevTools.
